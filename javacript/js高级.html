       通过跟踪调试发现，没有使用var定义的aMoney变量在创建过程与有var定义的变量有很大不同。

       1）隐式全局变量必须在执行完后，才会创建到全局域中【跟踪调试体验差异】。

       2）隐式全局变量aMoney，在执行后会创建到，全局变量域中。

 

 3）在全局域中有相同名称的变量，就不存在隐式全局变量的说法了，就是全局变量。

    aMoney = 10; //使用全局变量

          1.使用var定义的变量，在执行所在域时，就会创建这些变量，初始值为 undefined。

              2.如果没有使用var且在全局变量也没有定义，就会在执行后创建到全局域中，这样会给程序带来不稳定性，变量容易受到污染，不推荐使用。请养成好习惯，使用var定义变量。

闭包注意要点

1.由于大家对闭包的内存分配不是很清楚，主要表现在是不是使用了闭包，闭包的资源有没有被释放，例如 var  age = 10;，很多人认为这个变量会立即释放的。

2.建议使用赋null，请彻底清理闭包的资源，所以用完后 aFun=null;即可

3.由于闭包中的数据是共享的，很容易造成被改变，造成变量被污染。

4.闭包域的变量只创建一次，外面函数执行n次，就创建n个闭包与对象。

     function  counter() {

         var  arry =  new  Array();

         for  ( var  i = 0; i < 5; i++) {

            arry[i] =  function  () {

                console.log(i);

            }

        }

         return  arry;

    }

     var  array = counter();

执行了一次counter()方法，只创建一个闭包域 counter() ,闭包域的变量为变量i，变量i只声明一次，其余的都是对i的赋值操作而已。

 

问题：如何避免闭包的数据共享了？

 

3.5.3 防止闭包的数据共享

function  counter() {

     var  arry =  new  Array();

     for  ( var  i = 0; i < 5; i++) {

        arry[i] =  function  () {

            console.log(i);

        }

    }

     return  arry;

}

var  array = counter();

for  ( var  i = 0; i < array.length; i++) {

    array[i]();

}

结果输出的都是5，做如下分析

1.匿名函数和counter()函数之间构成了闭包，但所有的匿名函数对闭包作用域变量i是共享的。

解决方法

1.使用即时函数，为每个匿名函数形成一个私有的变量，解决变量共享的问题

function  counter() {

     var  arry =  new  Array();

     for  ( var  i = 0; i < 5; i++) {

        ! function  (j) { //使用即时函数定义私有变量j，再次创建一个闭包区

            arry[j] =  function  () {

                console.log(j);

            }

        }(i);

    }

     return  arry;

}

var  array = counter();

for  ( var  i = 0; i < array.length; i++) {

    array[i]();

}

4 原型链
4.1 什么是原型链：在js中，使用原型让一个引用类型继承另一个引用类型的属性和方法。
4.2 原型主要作用：实现对象之间的继承关系
4.3 如何采用原型链实现继承的？

一个空对象引发思考

var  aStudent = {};

console.log(aStudent);

通过开发工具可以看到对象aStudent对象的成员。

 

为什么一个空对象会有这么成员了？这些成员从哪里哪里来的了？

 

在aStudent对象里，系统自动给aStudent对象添加了一个__proto__的属性，这个属性指向一个构造函数为Object()的对象。

而Object对象就没有__proto__属性了，说明这个对象不再指向任何对象，是所有对象的根对象。

PS：在js的世界也是对象的世界，万物皆对象。和Java的OOP思想一致。在Java中使用extends实现继承，在JS使用对象的属性指向来达到异曲同工的效果。

问题：我们可以不可以自定义继承关系了？

自定义继承关系 

var  aObj = {

    name:  "Object A" ,

     root: null //指向父类对象

};

var  bObj = {

    address: "深圳市"

};

aObj.root = bObj;

console.log(aObj.name);

console.log(aObj.root.address);

console.log(aObj.address); //错误不能直接访问

 

     存在的问题：这个并具有真正意义上的继承特性，不能直接访问继承下来的属性或方法。

    对上面的代码做如下的改造：

var  aObj = {

    name:  "Object A" ,

     //当前对象，继承root对象

     extend:  function  (root) {

         for  ( var  va  in  root) { //循环遍历父类对象的属性和方法

             this [va] = root[va]; //对象之间的成员的拷贝。

        }

    }

};

var  bObj = {

    address:  "深圳市" ,

    show:  function  () {

        console.log( this .address);

    }

};

aObj.extend(bObj); // aObj 继承 bObj

    console.log(aObj.address); //直接访问继承下来的属性

aObj.show(); //直接访问继承下来的方法

    代码总结：以上代码通过extend方法可以很好的实现一个对象继承另一个对象的属性和方法。本质就是克隆对象，两个对象是互不影响的，如下图。

             根据前面原型链__proto__，可不可以通过它更加简单的实现？

var  aObj = {

    name:  "Object A" ,

};

var  bObj = {

    address:  "深圳市" ,

    show:  function  () {

        console.log( this .address);

    }

};

aObj.__proto__ = bObj;

console.log(aObj.address);

aObj.show();

使用原型链对象确实非常轻松的实现了aObj对象继承了bObj对象。

package servlet;

import java.io.File;

import java.io.FileOutputStream;

import java.io.IOException;

import java.io.InputStream;

import java.io.PrintWriter;

import java.util.List; 

import javax.servlet.ServletException;

import javax.servlet.http.HttpServlet;

import javax.servlet.http.HttpServletRequest;

import javax.servlet.http.HttpServletResponse; 

import org.apache.commons.fileupload.FileItem;

import org.apache.commons.fileupload.FileItemFactory;

import org.apache.commons.fileupload.FileUploadException;

import org.apache.commons.fileupload.disk.DiskFileItemFactory;

import org.apache.commons.fileupload.servlet.ServletFileUpload;

 

public class UploadServlet extends HttpServlet {

  private static final long serialVersionUID = 1L;

 

  public void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {

    doPost(request, response);

  }

  public void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {

 

    response.setCharacterEncoding("UTF-8"); // 设置处理请求的编码方式

 

    // 在Response头中指定Content-Type为text/html，而并非text/html，是可以解决问题的。这样返回给客户端的是一个JSON字符串（并非JSON对象）

    // ，无需IE来解析。随后，使用JSON.parse()函数将JSON字符串解析为JSON对象即可。

    // response.setContentType("application/json;charset=UTF-8"); // 设置Content-Type字段值

    response.setContentType("text/html;charset=UTF-8"); // 设置Content-Type字段值

    PrintWriter out = response.getWriter();

 

    // 实例化一个硬盘文件工厂，用来配置文件上传组件ServletFileUpload

 

    FileItemFactory factory = new DiskFileItemFactory();

    ServletFileUpload upload = new ServletFileUpload(factory); // 利用硬盘文件工厂配置文件上传组件

    List<FileItem> items = null; // 存放FileItem对象

    try { // 获取文件的FileItem对象，即表单域,分为普通表单域和文件域

       items = upload.parseRequest(request);

    } catch (FileUploadException e) {

       e.printStackTrace();

    }

    String path = null; // 存放上传文件的完整名称，包括路径。

    String filename = " "; // 存放文件名

    InputStream is = null;

    // 循环处理上传文件

    for (FileItem item : items) {

       if (item.isFormField()) {

         if (item.getFieldName().equals("filename")) {

           if (!item.getString().equals(""))

             filename = item.getString("UTF-8");

         }

       } else if (item.getName().trim() != null && !item.getName().trim().equals(" ")) {

         path = item.getName();// 得到文件完整路径

         filename = path.substring(path.lastIndexOf("\\") + 1);

         is = item.getInputStream(); // 获得上传文件的InputStream对象

       }

    }

    String pathName = request.getRealPath("/") + filename;

    if (new File(pathName).exists()) {

       // out.println("该文件已经存在，请为文件指定一个新的文件名！");

       out.println("文件已存在！");

    } else if (!pathName.equals("")) {

       FileOutputStream fos = new FileOutputStream(pathName);

       byte[] buffer = new byte[8192];

       int count = 0;

 

       // 开始读取上传文件的字节，并将其输出到服务器端的上传文件输 出流中

       while ((count = is.read(buffer)) > 0) {

         fos.write(buffer, 0, count); // 向服务器端文件写入字节流

       }

       fos.close();

       is.close();

         out.println("上传成功！"); 

    }

  }

}
